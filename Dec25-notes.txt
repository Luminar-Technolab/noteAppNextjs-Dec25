                CSS
        ------------------

    1. CSS (Cascading Style Sheets) : Styling html Elements
    2. What CSS is used for : Colors, fonts, spacing, Layouts (Flexbox, Grid), Responsive design (mobile, tablet, desktop), Animations & transitions, Dark/light themes
    3. Ways to apply CSS to html Elements
        - Inline CSS : using 'style attribute' of an html Elements
            - syntax : style="property:value;"
        - Internal CSS: using 'style tag' to apply css to html page
            - syntax :  <style>
                            css-selector {
                                property:value;
                            }
                        </style>
        - External CSS : using External CSS file to apply style to html Elements, link css file to html using 'link tag'
            - syntax : css-selector {
                                property:value;
                            }
    4. Selectors : used to select html Elements to css
        - using Tag name : syntax - tagname{ property:value;}
        - using id : syntax - #id{ property:value;}
        - using class: syntax - .class-name{ property:value;}
        - Universal selector : used to select all Elements in an html page, using *
        - CSS Combinators
            - Grouping selector : using commas
            - Descendant selector : using space
            - Child selector : using >
            - All Sibling selector : using ~
            - Immeadiate Sibling selector : using +
    5. Box Model : In webpage every HTML element is treated like a rectangular box.
        - The box model defines how space and size are calculated for that element.
        - Parts of the Box Model
            - Content : The actual content: text, image, video, etc., Width & height apply only to the content by default.
            - Padding : Space between content and border, Increases the visible size of the element
            - Border : Surrounds padding + content, Has thickness, style, and color
            - Margin : Space outside the border, Creates distance between elements
        - box-sizing Property
            - content-box : Width = content only
            - border-box : Width = padding + border + content
    6. Float : float moves an element to the left or right of its container and allows text or inline elements to wrap around it. 
    7. Flex Box : used to align, space, and arrange items in one direction (row or column)
        - Flexbox is a layout model for arranging elements inside a container tag: Horizontally or vertically, Even spacing, Easy centering (yes, finally!), Responsive by default
            - Flex container → the parent : set its display as flex
            - Flex items → the children
            - flex-direction : row , row-reverse, column , column-reverse
            - justify-content : Aligns items horizontally (row) or vertically (column)
            - align-items : Aligns items perpendicular to main axis , space to vertically
            - flex-wrap : Allows items to wrap into next line
            - gap : Spacing between items (cleaner than margin!)
    8. Position : controls how an element is placed on the page and what it’s positioned relative to.
        - position: static | relative | absolute | fixed | sticky;
        - when an element position is set then browser adds top, right, bottom & left position along with margin
    9. Psudeo-Class : A pseudo-class is a keyword added to a selector that defines a special state of an element.
        - syntax : selector:pseudo-class {   property: value; }
        - pseudo-class : hover/active/focus/visited/link/first-child/last-child/nth-child()/not/checked
        - Pseudo -class vs psudeo-element
            Pseudo-class → : (add specific state to an element)
            Pseudo-element → :: (used to style part of an element)
    10. transform : The transform property is used to rotate, scale, move, or skew an element in 2D or 3D space.
    11. transition : animate a property change smoothly over time (like hover effects).
            - syntax : transition: property duration timing-function delay;
    12. Grid : create complex, responsive layouts using rows and columns
        - to set up grid: use display : grid
        - To set up rows & columns use : grid-template-rows , grid-template-columns
    13. Linear Gradient : linear-gradient(side-or-corner, angle, color-stop1, color-stop2, ...);
    14. Animation in CSS : change CSS properties smoothly over time 
        - Key Parts of CSS Animation
            - @keyframes → defines the animation steps
            - animation → applies the animation to an element
    15. Examples of animation library : Animate css , aos animation, locomotive animation
    16. Responsive Website Design using 'Media Queries'
        - A media query lets you apply CSS only when certain conditions are true, like : screen size, device type, orientation (portrait / landscape)
        - different styles for different screens.
        - Basic syntax : @media condition {
                        /* CSS rules */
                        }
        - Common Breakpoints
            /* Small devices (phones) */
            @media (max-width: 576px) { }   
            /* Tablets */
            @media (min-width: 577px) and (max-width: 768px) { }
            /* Laptops */
            @media (min-width: 769px) and (max-width: 992px) { }
            /* Desktops */
            @media (min-width: 993px) { }

            ✅ Context API (Best built-in solution)

            MONGODB 
        -----------------

    1. MongoDB is NoSql , document-oriented db, that store data in JSON like format (BSON) instead of Table
    2. Features
        - Schema-less (flexible structure)
        - Scalable
        - Designed for modern app
    3. Key Features
        - Database : A container for Collections
        - Collection : Equalent to table in SQL
        - Document : A single record (JSON-like object) in collection
        - BSON : Binary version of JSON used internally
        - _id Field : Every document has a unique Key
    4. CRUD Operations
        - Read : To get documents from collection
            - find() : used to all documents, usage : collection-name.find(), to query from collection : find(condition), filter documents based on condition, condition is always written as object
            - findOne(condition) : used to get single document 
        - Create : To insert document into the collection
            - insertOne(document) : used to insert one document into a collection, document can be an object
            - insertMany([document1,doc2,...]) : used to insert multiple documents at a time into an collection
        - Update : To edit document from a collection
            - updateOne({filter} :  {update-query:{ field: value }} ) : used to update a single document
            - updateMany({ filter },   { $update-query: { field: value } }) : used to update multiple documents at the sametime
        - Delete : To remove document from a collection
            - deleteOne() : used to remove single document
            - deleteMany() : used to remove multiple documents

        - Comparison : syntax - {field:{comparison-query:value}}

        - Logical : syntax - {logical-query:[{field1:value1},{field2:value2}]}
            - ex: $and, $or, $nor, $not
        - Evaluation : syntax - {field:{evaluation-query:value} }
            - ex: $regex, $text, $where, $options
    6. Additinal Operations : count(), sort(), limit()
    7. Update operators 
        - Field : $inc, $set, $unset, $rename, $currentDate
        - Array : $pop, $pull, $addToSet, $push
    8. Indexing : used to make data searching faster in MongoDB.

        - Syntax : db.collection.createIndex({ fieldName: 1 })
    9. Schema Design : Structure of Collection
        - Embedded Schema Design : add entire document details into it
        - Referenced Schema Design : using Primary & Foreign Key for reference
    10. Aggregation : used to process and transform data inside db, It is used for reports, analytics, filtering, grouping, sorting.
        - Syntax : db.collection.aggregate([    { stage1 },    { stage2 } ])
        - MongoDB uses pipeline stages. $match, $group , $sort, $project, $limit, $skip, $lookup
        - $lookup: to join collections

    next-auth / Auth.js Package
    ----------------------------

    1. next-auth is an authentication library used in Next.js applications to handle: Login / Signup, Social authentication, JWT & session management, Route protection, OAuth providers, Credentials-based authentication
    2. Installation : npm install next-auth
    3. Main Features :
        - Authentication Providers : Google, GitHub,Facebook,Credentials Login etc.
        - Basic Folder Structure (App Router)
            app
            └── api
                └── auth
                    └── [...nextauth]
                            └── route.ts
    4. next-auth Working
        - user click login btn using email/password form
        - Requst goes to 'authentication provider' of next-auth
        - Authentication provider used to verify user. 
        - If authentication provider return true then Sesion/JWT is created
        - Session should be stored in client part depending on strategy. JWT stored in browser cookie, db session in db. To view the session use http://localhost:3000/api/auth/session
        - Access user data anywhere using hooks/function

        Frontend UI --> signIn() --> nextAuth API Route --> Provider Authentication --> Session/JWT created --> Protected Page Access
    5. Important Functions
        - signIn() : login
        - signOut() : logout
        - useSession() : client component can access session
        - getServerSession() : server component can access session
        - SessionProvider : Global auth state
